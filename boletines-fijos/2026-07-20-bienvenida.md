El ecosistema del dato se mueve demasiado rápido.

Aparece una nueva versión de Python. Microsoft anuncia una nueva funcionalidad para Power BI. DuckDB incorpora una mejora. OpenAI presenta un nuevo modelo.

Durante unas horas, todo el mundo habla de ello. Una semana después, casi nadie lo recuerda. Y seis meses más tarde, resulta sorprendentemente difícil responder a una pregunta tan simple como:

> ¿Cuándo ocurrió aquello?

Ahí nació la idea de EVOLUCIÓN.

No quería crear otro boletín de noticias, ni tampoco otro blog sobre Data Analytics. Quería construir lo que yo misma echaba de menos: un lugar donde seguir la evolución de este ecosistema sin depender del algoritmo de una red social ni perderme entre cientos de publicaciones.

EVOLUCIÓN también nace de una necesidad personal. Cuando empecé a estudiar Data Analytics descubrí que la mejor forma de aprender era investigar, resumir, organizar y escribir. Lo que comenzó como un hábito para mantenerme actualizada terminó convirtiéndose en este proyecto.

Cada edición añade una nueva pieza a este archivo. No importa si hoy hablamos de SQL, Microsoft Fabric, Databricks o Inteligencia Artificial. Lo importante es que, dentro de cinco años, siga siendo posible entender cómo evolucionaron.

> Las noticias son instantáneas. La evolución solo se comprende cuando alguien decide documentarla.

Para ello, cada edición se elabora a partir de fuentes oficiales, documentación técnica, blogs de fabricantes de software y medios especializados. Siempre que es posible, cada noticia incluye un enlace a la fuente original para que cualquier lector pueda consultar la información directamente desde su origen.

Cada edición pasa a formar parte de un archivo permanente cuyo propósito es documentar, con rigor y fuentes verificables, la evolución del ecosistema del dato en español.

Si hoy has llegado hasta aquí, gracias.

Quizá mañana vuelvas para leer una nueva edición. O quizá regreses dentro de unos años para consultar cómo evolucionó una tecnología concreta.

En cualquiera de los dos casos, este archivo seguirá creciendo.

> Una edición cada día. Una evolución a la vez.
